# MVP Deferred Scope (Explicit)

The following items are intentionally deferred and excluded from this MVP:

- Leave requests and approvals
- Shift scheduling and break enforcement
- Core-hour validation
- Attendance exception management beyond invalid/duplicate punch handling
- Overtime, holiday, night differential, PTO, comp-time calculations
- Full compliance reporting and CrossCheck reporting
- Multi-company and multi-location depth
- Full RBAC model and production-grade authentication
- CIAM integration and event-driven architecture implementation
- Full support and BugBot runtime implementation

## Deferral Principle
If a feature does not directly improve the MVP demo path (clock-in, validation, clock-out, history, audit, payroll summary), it stays deferred.

## Post-MVP Expansion Note

The first approved post-MVP expansion slice is missing punch detection (EXP-A1 to EXP-A3). All other deferred items remain deferred until that slice is stable.
